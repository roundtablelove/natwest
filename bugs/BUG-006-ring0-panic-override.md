# BUG-006 — Ring 0 Panic Override Produces False Report

```
      ID: BUG-006
Severity: CRITICAL
  Module: Branch Manager — Wetware Panic Response
  Status: CONFIRMED
    Date: 2021-09-21 ~09:00 UTC
Location: NatWest, Brixton Road, London
```

## Summary

When primal fear took over, the manager's organism called for backup regardless
of whether the threat was active. The police became an extension of one man's
panic response — Babylon's dogs dispatched on the basis of a frightened man's
ring 0, not on the basis of evidence.

## Mechanism

Prison collapses the processing stack. Every higher function — planning,
abstraction, strategic thinking — is a luxury that depends on safety. The
customer was 2 days out of prison after ten days on remand (false charges). He
was operating without a persona layer. No customer-service smile. No filter.
Raw.

The manager's firmware has a matching failure mode. When primal fear fires:

1. Higher cognitive functions (assessment, context, proportionality) go offline.
2. Ring 0 seizes command — pure survival firmware.
3. The organism executes its only available defensive action: summon backup.
4. The content of the backup request is generated by the fear state, not by
   observed facts.

The result is a 999 call whose content reflects the manager's internal terror,
not the customer's actual behaviour. The customer made no threat. Did not move
a muscle. He showed the hint of a tooth and called the manager an arsehole. The
manager terrified himself.

## Impact

Ring 0 fear produces false reports. The 999 recording (pending SAR) will show
what the manager reported. The customer's videos show what actually happened.
The delta between the two is the measure of this bug.
